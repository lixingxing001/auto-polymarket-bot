# Execution Backtest Report

## 本轮结果

最近 48 个窗口的真实成交代理回测结果：

- 已结算样本：47
- 因成交历史不完整跳过：1
- 因优势不足跳过：7

- 胜率：30.8%
- 总 PnL：-15.36 USDC
- 平均每笔 PnL：-0.39 USDC

## 关键发现

1. 仅凭外部价格特征得到的概率预测，无法直接和 Polymarket 报价做套利比较。
2. 当前模型经常把低价结果误判成“便宜货”，实际买到的是市场已经明显不看好的一侧。
3. 这说明 Polymarket 自身价格必须进入模型，至少作为：
   - 强基线
   - 输入特征
   - 回测对照组

## 为什么这一步重要

方向准确率只回答“猜得对不对”。

可执行价格回测回答：

- 当时能不能成交
- 成交价是多少
- 扣费后还有没有优势
- 下注后到底赚不赚钱

这一步已经证明，当前机器人离实盘还有明显距离。

## 下一步建议

1. 接入历史 `Up / Down` 价格，构建 market-aware 数据集
2. 先做一个很朴素的基线：
   - 直接跟随 Polymarket 当时更贵的一侧
3. 再训练混合模型：
   - Polymarket 价格
   - Coinbase 价格行为
   - 微结构特征
4. 只接受在样本外同时满足以下条件的模型：
   - 胜过 Polymarket 价格基线
   - 扣费后 PnL 为正
   - walk-forward 仍然稳定
